# Strategy Variations

This folder stores controlled strategy variations for offline backtests only.

Rules:
- Do not modify immutable core constraints from `strategy/core-rules.md`.
- Create one variation per file.
- Change one parameter family at a time.
- Record expected impact before testing.

Recommended file naming:
- `V001-drop-threshold-adjustment.md`
- `V002-zscore-threshold-adjustment.md`
- `V003-tp-sl-balance.md`
